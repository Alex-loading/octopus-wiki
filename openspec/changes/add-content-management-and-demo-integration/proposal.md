## Why

当前项目是纯前端静态内容，文章与技术 demo 以代码内数据维护，导致内容持久化不足、更新流程依赖开发者改代码、演示内容难以系统化管理。现在需要引入可持续的内容管理能力，以支撑后续持续发布与扩展。

## What Changes

- 新增文章数据库存储能力，支持文章元数据与正文的结构化持久化。
- 新增便捷的文章更新流程能力，降低非开发者更新文章的门槛（支持标准化提交流程与发布校验）。
- 新增技术 demo 集成能力，支持 demo 条目管理、展示与与文章互链。
- 为前端读取层增加统一内容访问接口，替换仅依赖本地静态数据文件的模式。

## Capabilities

### New Capabilities
- `article-storage`: 定义文章在数据库中的存储模型、读写接口与发布可见性规则。
- `content-update-workflow`: 定义文章创建、编辑、校验、发布的标准化更新流程与操作约束。
- `demo-integration`: 定义技术 demo 的数据模型、展示入口、与文章关联规则。

### Modified Capabilities
- （无）

## Impact

- 影响代码：`src/app/data/posts.ts` 的内容来源将逐步迁移到服务端/数据库接口；`src/app/pages/Blog.tsx`、`src/app/pages/Post.tsx`、`src/app/pages/Lab.tsx` 的数据加载方式将调整。
- 影响系统：新增数据库与内容管理后端（可采用 BaaS 或自建 API 层）。
- 影响流程：内容更新从“改代码发版”演进为“内容入库 + 前端自动读取”的发布方式。
- 影响依赖：可能新增数据库 SDK、内容校验工具、CI 发布校验步骤。
