## Why

当前站点文章已迁移到数据库读取，但缺少可直接执行内容运营的管理入口。文章上传、删除、修改仍依赖工程手工操作，导致发布效率低、误操作风险高、内容变更不可审计，因此需要基于 Supabase 增加轻量后台管理能力，并与主站视觉风格保持一致。

## What Changes

- 新增面向内容管理员的文章管理后台页面，支持文章创建（上传）、编辑、删除与状态管理（草稿/发布）。
- 新增文章管理 API/数据访问层（基于 Supabase），统一封装写操作与错误处理。
- 新增后台鉴权与权限控制（仅授权管理员可进行写操作）。
- 新增文章管理流程约束：删除前确认、字段校验、保存后反馈、失败可回退提示。
- 新增后台 UI 规范：沿用主站设计 token、组件风格与交互语言，保证一致性。

## Capabilities

### New Capabilities
- `article-admin-console`: 定义文章后台管理页面的信息架构、列表与编辑交互、表单与操作反馈。
- `article-write-workflow`: 定义文章创建、修改、删除的写入规则、校验与异常处理行为。
- `admin-authz`: 定义后台访问鉴权、管理员权限边界与未授权处理规则。

### Modified Capabilities
- （无）

## Impact

- 影响前端代码：新增管理端路由与页面，扩展 `src/app/content` 写接口，复用现有 UI 体系。
- 影响数据库与权限：需在 Supabase 侧补充 RLS / policy 或 service role 调用边界。
- 影响运营流程：内容更新从“改代码”切换为“后台可视化管理 + 受控发布”。
- 影响依赖与安全：可能引入富文本/Markdown 编辑依赖、上传能力与鉴权中间层。
